Graph-Based RAG Summarization

October 2025 – Present

Project Overview

This is my final project for the Advanced NLP class at UMass Amherst. Together with a teammate, I am building a retrieval-augmented generation (RAG) system to summarize long meeting transcripts from the QMSum dataset. We first set up standard RAG baselines with BM25 and Contriever, then add a graph-based retriever using the Graph of Records (GoR) idea to improve the retrieval step.

We compare these setups by looking at ROUGE scores and by using an LLM judge to check which text chunks are retrieved and how they affect the final summaries. I like this project because it connects a research idea (using graphs in retrieval) to a system that has to work end to end on real data.

  • Built a retrieval augmented generation (RAG) pipeline for long meeting summarization on QMSum, comparing sparse BM25, dense Contriever, and Graph of Records (GoR) retrievers on FAISS indexes.
  • Evaluate summary quality with ROUGE and analyze retrieved chunk quality with an LLM judge to refine chunking, retrieval strategies, and prompts.
